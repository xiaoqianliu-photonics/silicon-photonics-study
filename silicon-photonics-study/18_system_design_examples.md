# 18. 系统设计实例与架构演进 (System Design Examples)

## 1. 学习目标

* 将本书前 17 章零散的“器件积木”拼装成完整的商业级光通信系统。
* 深刻理解光收发机（Transceiver）中 Tx（发射）和 Rx（接收）链路的全貌。
* 通过分析 100G PSM4 和 400G DR4 等真实商业架构，掌握工业界在并行与复用（Parallel vs. Multiplexed）之间的架构权衡。
* 了解相干通信（Coherent Communication）如何在长距离中突破速度极限。
* 开拓视野：硅光不仅用于通信，更是 LiDAR（激光雷达）、光计算和生物传感器的基础硬件平台。

## 2. 背景与直觉

你现在已经拥有了一个包含波导、滤波器、调制器、探测器和光栅耦合器的武器库，也知道了流片、工艺变异和链路预算的残酷现实。现在，我们要用这些武器来打一场真正的仗：**搭建一个能在数据中心里卖出钱的高速光模块**。

系统设计（System Design）本质上是在玩一个“走钢丝”的游戏。如果你要传输 400 Gbps 的数据，但目前一个调制器最快只能跑 100 Gbps，你怎么拼出 400G？
有两个直觉的选择：
1. **多修几条路（空间并行，Space Division）**：用 4 根光纤，每根跑 100G。这叫并行架构（如 PSM4）。
2. **不同颜色的车跑同一条路（波分复用，WDM）**：用 4 个不同波长的激光，复用到 1 根光纤里。这叫 WDM 架构（如 CWDM4）。

系统架构师每天的噩梦，就是在这两种方案的成本、功耗、芯片面积和光纤布线成本之间做选择。

## 3. 核心概念

| 英文术语 | 中文解释 | 物理意义/背景含义 | 设计中为什么重要 |
| ---- | ---- | ---- | -------- |
| Transceiver (TRx) | 光收发机 / 光模块 | 同时包含发射端（Tx）和接收端（Rx）的完整独立模块。 | 数据中心网络互连的基本单元。 |
| Parallel Single Mode (PSM) | 并行单模架构 | 使用多根平行的光纤传输多路独立信号，每路信号波长相同。 | 不需要波分复用器，芯片最简单，但光纤电缆很贵。 |
| Coarse WDM (CWDM) | 粗波分复用 | 通道波长间隔很大（如 20 nm）的波分复用。 | 对激光器波长漂移不敏感，通常不需要 TEC 温控，功耗低。 |
| PAM4 (Pulse Amplitude Modulation) | 四电平脉冲幅度调制 | 不仅仅用“亮”和“暗”代表 0 和 1，而是用 4 种亮度代表 00, 01, 10, 11。 | 在器件带宽不变的情况下，将数据传输速率翻倍的核心技术。 |
| Coherent Communication | 相干通信 | 不仅调制光的“强度”，同时调制光的“相位”和“偏振”，并在接收端与本地振荡激光干涉解调。 | 频谱效率极高，能在长距离海底或城域网传输极高数据率（如 800G+），但 DSP 极其耗电。 |

## 4. 系统架构剖析

### 4.1 典型的高速 Tx（发射）链路
以单通道 100G 为例：
* **数据源**：交换机芯片输出 100G 的电信号。
* **电学驱动（Driver/DSP）**：极弱的电信号需要被射频驱动芯片放大，达到驱动调制器所需的电压幅度（如 2Vpp）。
* **光输入**：外部大功率连续激光器（CW Laser）提供“长明灯”光源。
* **调制 (Modulator)**：硅光 MZM 或微环，根据电信号把光快速“斩断”，刻上 100G 的数据。
* **复用 (如果使用 WDM)**：AWG 或级联 MZI 把多个波长合为一束。
* **光输出**：光栅或端面耦合器将光送入输出光纤。

### 4.2 典型的 Rx（接收）链路
* **光输入**：带有数据的弱光从光纤进入芯片。
* **解复用 (如果使用 WDM)**：滤光片把混合的光按照颜色拆分到不同波导。
* **探测 (Photodetector)**：锗硅 PIN 探测器吸收光信号，吐出微弱的光电流（几百微安级别）。
* **电学放大（TIA）**：极其关键的“跨阻放大器 (Transimpedance Amplifier)”，把微弱电流放大为几百毫伏的电压，并送入数字信号处理（DSP）芯片进行纠错（FEC）和恢复。

## 5. 真实工业界案例对抗：100G PSM4 vs CWDM4

为了达到 100 Gbps ($4 \times 25$ Gbps)：

### 阵营 A：100G PSM4 (Parallel Single Mode 4-lane)
* **架构**：1 个激光器（分成 4 份），4 个相同的调制器，4根发射光纤 + 4根接收光纤。
* **优势**：芯片上没有任何滤波器，完全不用担心工艺误差导致的波长漂移。极其鲁棒，不需要温控。
* **劣势**：光纤太费了（8根光纤）。如果你要在两个机柜之间拉 500 米的线缆，8 芯光缆的成本极其昂贵。**主要用于短距离（<500m）互连。**

### 阵营 B：100G CWDM4 (Coarse WDM 4-lane)
* **架构**：4 个不同波长的独立激光器，4 个对应的调制器，1 个 4-channel 的 MUX，1根发射光纤 + 1根接收光纤。
* **优势**：极大节省了光纤成本（仅需 2 根光纤），**非常适合中长距离（2km - 10km）互连。**
* **劣势**：芯片极度复杂。需要四种不同波长的昂贵激光器。MUX/DEMUX 带来的插损高达几 dB，而且如果工艺波动导致滤波峰漂移了，串扰会毁掉系统。

**400G 时代的演进**：当单通道速率突破 100G 时（通过采用 PAM4 技术），行业内衍生出了 **400G DR4** 等标准，本质上仍然是在空间并行和波分复用之间寻找系统成本的最低点。

## 6. 超越通信：硅光子学的未来版图

除了做光模块，本书介绍的所有器件正在颠覆其他赛道：

1. **LiDAR (激光雷达)**：
   * 将成千上万个光栅耦合器排布成 2D 阵列，通过改变每个光栅上的相移器（热或电），实现相控阵 (Optical Phased Array, OPA)。不需要任何机械旋转，就能像雷达一样用光束扫描四周，是自动驾驶的关键技术。
2. **光计算 / AI 加速器 (Optical Computing)**：
   * 用级联的 MZI 阵列（或微环）在模拟光域里直接做矩阵乘法 ($Y = W \cdot X$)。因为光信号不需要等待电容充电，可以在皮秒级别完成计算且功耗极低，被视为打破摩尔定律、解决大模型算力瓶颈的颠覆性方案。
3. **量子计算与光量子技术 (Quantum Photonics)**：
   * 利用非线性干涉效应（如纠缠光子对源），硅光网络能在芯片上操纵量子比特，实现极高稳定性的量子线路。

## 7. 全书总结寄语

经过 18 章的跋涉，你已经完成了从基础物理到系统架构的硅光子学通关之旅。
硅光子学是一门充满残酷折中（Trade-off）的工程艺术：**追求速度就必须忍受损耗（等离子体色散的诅咒），追求高集成度（微环）就必须与热学漂移和工艺误差作斗争。**
但正是由于 CMOS 大规模制造的无穷威力，硅光正在一步步接管电子学的地盘，从海底光缆，到数据中心，再到未来的 GPU 芯片互连。欢迎来到光互连的时代。

## 8. 本章知识图谱

* 硅光系统架构与应用
  * 光收发机 (TRx) 结构
    * Tx：驱动放大 $\rightarrow$ 激光源 $\rightarrow$ 调制 $\rightarrow$ (复用)
    * Rx：(解复用) $\rightarrow$ 探测 $\rightarrow$ 跨阻放大 (TIA) $\rightarrow$ DSP
  * 扩容技术路线权衡
    * 空间并行 (PSM4, DR4)：省芯片，费光纤。适用短距。
    * 波分复用 (CWDM4, LR4)：费芯片，省光纤。适用中长距。
    * 高阶调制 (PAM4)：压榨单通道极限带宽。
    * 相干通信 (Coherent)：极限频谱效率，长距霸主。
  * 扩展应用领域
    * OPA 相控阵激光雷达 (LiDAR)
    * 光学矩阵计算 (AI 加速)
    * 片上光量子计算

## 9. 复习问题

**基础问题**
1. 在光收发机中，Tx（发射）和 Rx（接收）链路分别执行什么最核心的光电转换任务？
2. PSM4 架构与 CWDM4 架构最核心的区别是什么？这两种架构分别试图优化掉系统成本中的哪一部分（光纤还是光学芯片）？
3. PAM4 调制与传统的 NRZ（只发送 0 和 1）调制相比，在物理上有何区别？它如何在不提高器件带宽的情况下提升数据率？
4. 在接收端，探测器输出的光电流通常极其微弱，我们需要什么极其重要的电子放大器件来抢救这个信号？
5. 除了光纤通信，硅光子学还能用于哪些前沿领域？（列出两个）

**进阶问题**
1. 在相干通信（Coherent）系统中，除了探测光信号的强度，还需要探测其相位。这在芯片架构上要求必须引入本振激光器和复杂的干涉结构（如 $90^\circ$ 桥）。请简述为什么相干技术虽然极其复杂且昂贵，但依然主宰了数百公里的海底长途光缆市场？
2. 在 400G DR4 系统中（4 条并行的 100G 通道），如果采用一个超大功率的连续波（CW）激光器分成 4 份供给四个调制器，这样做在链路预算和热管理上会面临什么严重的隐患？
3. 如果将硅光芯片用于光学神经网络加速器，其核心是在模拟光域完成矩阵乘法。请从延时和功耗的角度，解释光学矩阵计算相较于传统数字 GPU 架构的理论优势。

**开放式设计问题**
1. 假设你是一流数据中心（如谷歌或亚马逊）的网络架构总监。随着未来 AI 集群规模的爆炸，交换芯片的背板 I/O 带宽即将耗尽，传统的插拔式光模块变得极其臃肿且功耗巨大。请你设计并论述一种“共封装光学（CPO, Co-Packaged Optics）”架构方案。你会把硅光芯片放在哪里？外部激光源（ELS）放在哪里？在这个疯狂的高密度集成方案中，你认为最具毁灭性的技术挑战是什么（考虑热学、良率和光纤对准）？
