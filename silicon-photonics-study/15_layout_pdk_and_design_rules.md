# 15. 版图设计、PDK 与设计规则 (Layout, PDK and Design Rules)

## 1. 学习目标

* 明白物理设计（Layout）在硅光子学中为什么比传统微电子学更加“反人类”（全是不规则曲线）。
* 掌握 PDK（工艺设计套件）的核心构成，了解它如何拉平设计公司与代工厂之间的沟通鸿沟。
* 理解 DRC（设计规则检查）背后的物理与制造逻辑（为什么不能把两条线画得无限近）。
* 了解 LVS（版图与原理图一致性检查）在光子学中的现状与挑战。
* 能够区分 Schematic-driven Layout (SDL) 与手动 GDS 绘制的区别。

## 2. 背景与直觉

在仿真软件里，你的波导是绝对平滑的理想几何体。但代工厂的机器不认识你的 FDTD 模型文件，它们只认一种东西：**GDSII 掩模版图**（或者叫 GDS 文件）。这就像是给建筑工人的“施工图纸”，上面用不同的颜色层画满了你要在晶圆上刻蚀或保留的二维多边形。

麻烦在于，微电子芯片的线都是横平竖直的（因为晶体管是方的，金属走线是曼哈顿布线）。但**光怕拐直角**，硅光芯片里全是圆弧（弯曲波导）、斜线（Taper）甚至同心圆（光栅）。这让画版图变成了一项极其痛苦的工作。
更要命的是，你不能随心所欲地画。代工厂的刻蚀机和光刻机有物理极限，如果你把两根波导画得太近（比如只有 20 nm），光刻机根本分辨不出来，两根线最后会糊在一起。这就引出了一个所有流片人都必须遵守的宪法：**设计规则 (Design Rules)** 和执行它的工具 **PDK (工艺设计套件)**。

## 3. 核心概念

| 英文术语 | 中文解释 | 物理意义/背景含义 | 设计中为什么重要 |
| ---- | ---- | ---- | -------- |
| PDK (Process Design Kit) | 工艺设计套件 | 代工厂提供给设计者的“零件库”与“说明书”。包含标准器件的版图、仿真模型和检查规则。 | 使用 PDK 中的标准化器件（如 1x2 MMI），可以免去重复仿真，保证一次流片成功率。 |
| GDSII (GDS) | 图形数据系统 | 半导体工业最基础的版图文件格式。所有的器件最终都会被转换成无数个二维多边形。 | 是你发给代工厂流片的唯一凭证。 |
| Design Rules | 设计规则 | 代工厂规定的物理尺寸红线。比如最小线宽 (Min Width)、最小间距 (Min Space)。 | 确保你的设计在现实中“能被刻出来”且“不短路/不断路”。 |
| DRC (Design Rule Check) | 设计规则检查 | 提交 GDS 前的软件强制检查。一旦报错，代工厂将直接拒收你的设计。 | 避免因为你的低级设计失误浪费昂贵的掩模版和晶圆。 |
| LVS (Layout Versus Schematic) | 版图原理图一致性检查 | 检查你画的 GDS 版图，和你在脑子里/电路图里的连接逻辑是否完全一致。 | 确保你没把波导连错端口，但目前在硅光领域 LVS 技术还不像微电子那么成熟。 |

## 4. 关键原理：PDK 的剖析

一套合格的代工厂 PDK（如 AIM Photonics, IME, TowerJazz）包含三部分：

### 4.1 技术文件与层映射 (GDS Layer Map)
* 在 GDS 文件中，数字“层（Layer）”代表不同的物理制造步骤。
* 例如：Layer 1 可能是全刻蚀（刻穿 220nm 硅），Layer 2 可能是浅刻蚀（刻下 70nm 留作光栅），Layer 31 可能是 P 型轻掺杂，Layer 41 可能是顶层金属（Al）。
* 你必须严格按照代工厂的 Layer Map 给你的多边形分配图层。如果用错了层，你的波导可能被当成铝线制造。

### 4.2 标准器件库 (Component Library)
* 包含已经过硅片验证（Silicon-verified）的“黑盒”器件。比如标准的 TE 偏振光栅耦合器、标准的 50:50 MMI。
* 代工厂不仅提供这些器件的固定 GDS 版图，还会提供它们的 **S参数模型**，供你在系统仿真（如 Lumerical INTERCONNECT）中直接调用。

### 4.3 设计规则参数
* **Minimum Space (最小间距)**：光刻胶在狭窄缝隙中难以被显影液洗掉，刻蚀不透，导致本该分开的波导粘连（Bridging）。通常在 100nm - 200nm。
* **Minimum Width (最小线宽)**：线太细，刻蚀时容易断裂甚至被完全冲掉。
* **Density Rules (密度规则)**：整个芯片的硅面积和空白面积必须保持在一个固定比例（例如 30%-70%）以防 CMP（化学机械抛光）时把硅片磨得高低不平。这就是为什么你经常在别人版图的空白处看到一堆无意义的方形硅块（Dummy Fills）。

## 5. 工程流程：如何从想法到 GDS

### 5.1 传统方法：基于多边形的手动绘制 (Scripting / Polygon-pushing)
* 使用 KLayout, GDSPY (Python) 或 Nazca 等工具，通过写代码的方式把坐标一个一个敲进去，画出圆弧和多边形。
* **优点**：极致的自由度，适合学术界探索奇形怪状的物理器件。
* **缺点**：系统一大，代码维护简直是地狱；波导稍微挪动一下，所有的连接曲线全部要重新算数学方程。

### 5.2 现代方法：原理图驱动版图 (Schematic-driven Layout, SDL)
* 类似于画 PCB 板。先在 Virtuoso 等软件中画一个原理图（这儿放一个微环，那儿放一个调制器，用线连起来）。
* 软件自动根据原理图，在版图界面调出对应的 PDK 器件，并**自动帮你计算弯曲半径，自动布线 (Auto-routing)** 把光波导连好。
* **优点**：不易出错，效率极高，适合上百个器件的大型商业芯片。

## 6. 常见误区

* **误区：既然定向耦合器的 Gap 越小耦合越强，那我就在 GDS 里画个 50nm 的 Gap，这样器件能做到最小。**
  * *纠正*：你马上就会在 DRC 检查时被代工厂按在地上摩擦。标准深紫外光刻 (193nm DUV) 的极限通常在 100nm 到 150nm 左右（除非采用极其昂贵的浸没式光刻或电子束光刻 EBL）。违反 DRC，晶圆厂会直接拒单。
* **误区：只要过了 DRC 检查，芯片就一定能正常工作。**
  * *纠正*：DRC **只检查形状，不检查物理**。如果你的两条波导虽然相距很远满足了 DRC，但你把发射端波导连到了探测器的错误端口上，DRC 是不会报错的。你的芯片依然是个废品。这就是为什么我们需要 LVS 检查。

## 7. 本章知识图谱

* 流片接口与版图设计
  * GDSII 文件 (通用二维几何图形)
  * 工艺设计套件 (PDK)
    * 图层映射 (Layer Map)
    * 硅验证的器件库 (S参数 + 固定版图)
  * 设计规则检查 (DRC)
    * 最小线宽 / 最小间距
    * 密度规则 (加 Dummy fills 防 CMP 抛光不均)
  * 物理布线 (Routing)
    * 不支持曼哈顿直角布线，必须考虑弯曲半径和绝热过渡
    * SDL (原理图驱动版图) 自动化提升效率

## 8. 复习问题

**基础问题**
1. PDK 在芯片设计公司（Fabless）和代工厂（Foundry）之间扮演了什么角色？
2. 什么是 DRC 检查？举出两个常见的光子学设计规则。
3. 为什么在画金属线时可以随意画 90 度直角，而在画光波导时必须使用大圆弧？
4. GDSII 文件里包含电磁场仿真的折射率信息吗？它是用来干什么的？
5. LVS 的主要目的是什么？

**进阶问题**
1. 有些设计要求在微环内部或者周边留出大片完全空白的区域，但在提交流片前，软件强制你在这些空白区填满细碎的“假正方形（Dummy Fills）”。请从制造工艺（CMP）的角度解释为什么必须加这些废料？
2. 在版图布线时，两根平行的直波导距离太近会发生什么？这违反了 DRC 还是违反了光学隔离法则？
3. 如果你自己用 3D FDTD 设计了一个性能逆天的奇特形状分光器，但它有一处的尖角只有 50 nm，违反了代工厂的最小宽度 DRC 规则。你该如何说服自己妥协并修改设计？

**开放式设计问题**
1. 在传统 CMOS 电子布线中，两根导线如果需要交叉，只要放在不同的金属层（比如 Metal 1 和 Metal 2）就不会短路。但是在 SOI 硅光子学中，导光层通常只有薄薄的一层 220nm 硅。如果系统极其复杂，波导必须发生交叉（Waveguide Crossing），你会在版图上使用什么器件？设计这个器件时，你需要解决哪些光学物理问题来避免串扰和损耗？
